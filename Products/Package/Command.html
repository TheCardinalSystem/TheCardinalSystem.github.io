import com.Cardinal.CommandPackage.Commands.ICommand;
import com.Cardinal.CommandPackage.Exceptions.MissingArgumentsException;

import sx.blah.discord.handle.impl.events.guild.channel.message.MessageReceivedEvent;

public class CommandPing implements ICommand {

	public CommandPing() {
	}

	@Override
	public String getName() {
		return "ping";
	}

	@Override
	public String getDescription() {
		return "pong!";
	}

	@Override
	public void execute(MessageReceivedEvent event, String[] args) throws MissingArgumentsException {

		event.getChannel().sendMessage("Pong!");

	}

}


Registering a command is done as follows:
1
2
3
//import com.Cardinal.CommandPackage.Proccessor.CommandRegistry;
//Command is of type ICommand
CommandRegistry.registerCommand(command);


Users may create and share commands by exporting subclasses of
ICommand into Jars (command bundles are supported).

When adding these bundles/commands, just copy the jar file to the
'Commands' folder that is generated and call the following method:

CommandLoader#startCommandLoader();




		
If a command package contains commands you do not wish to implement, just call the following method:


CommandRegistry#unregisterCommand("commandName");
